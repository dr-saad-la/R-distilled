Installing Classic Jupyter Notebook, Jupyter Lab and IRkernel

Introduction#

  • In this section, we will install Python so we can install Jupyter notebook, it is not required for R, it is just for installing jupyter notebook.

  • If you already have Python, you can skip this section, if you are not sure, you can check Python by starting the command prompt and type python --version

  • If you by any chance have anaconda distribution installed, you already have the necessary tools and you can move to the next section of installing IRkernel.

Installing Python: Prerequisite for Installing Jupyter#

  • Python is a requirement for installing the Jupyter Notebook. After installing Python, the Python package manager pip will be installed automatically, which is the tool we use to install Jupyter notebook or Jupyterlab.

  • The Python version required differs between Jupyter Notebook releases (e.g. Python 3.6+ for Notebook v6.3, and Python 3.7+ for Notebook v7) . source

Installing Classic Jupyter Notebook#

  • First let us ensure that you have pip Python package manager by running:

pip --version

  • If it does not produce an error, it means your machine is set up correctly, if not, you have to follow these steps to add Python to the path (the path is the location where the system searches for executable files):

  • Adding Python to the path:

    • You can do this either from the command prompt as follows:

      • Open command prompt in administrator mode (press windows key + R), then type cmd in the Run dialog, then press ctrl + Enter

      • Then just type this command setx /m "python-path:%PATH%"

      • replace python-path with the location of Python on your system

      • Close up the command prompt and open it again

      • type python --version

      • If returned Python version, then Python has been added to the path correctly.

    • Adding Python to the path using graphical user interface:

      • Open the run dialog (this is the easiest way)

      • Type sysdm.cpl in the Run dialog and press enter

      • The System properties will pop up

      • Go advanced tab

      • Press on Environment variables

      • Go to System variable and find path and press on edit (This will add Python system wide)

      • Press on new

      • Add the location of the executable Python, this location looks like this C:\Program Files\Python3.x\scripts

      • Press ok

      • You can add Python to this specific user the same way you did with system variable from User variables

      • When you finish, press ok, the ok from the environment variables.

      • Test that Python has been added to the path from the command prompt python --version.

  • If you followed the steps above of installing Python and making sure it is recognized from the command prompt, then you are good to go, otherwise you should go up the steps illustrated above.

  • We can install the classic Jupyter notebook by running the following command:

pip install jupyter

or

pip install notebook

  • You can test that jupyter is installed by running the next command:

jupyter --version

  • This command will show information jupyter version installed with extra information that it will be clear in a later stage.

Launching Jupyter Notebook#

  • My preferred way to start jupyter notebook is from the command prompt, however, you can start it however you like. The reason I start jupyter from the CLI is that I can change the path the desired directory first then start this application from there instead of starting it from the default location of CLI.

  • Here are the steps of launching Jupyter:

    1. Open the command prompt (or any command line you have)

    2. Change the directory to your preferred working directory using this command cd "path-to-your-working-directory"

    3. Launch Jupyter by typing jupyter notebook

    4. Jupyter will open in your default browser.

Installing JupyterLab#

  • JupyterLab is the newer version of notebook, it is recommended to use this version. You can install it similarly to classic notebook by running the next command:

pip install jupyterlab

  • Test that jupyterlab is setup correctly using this command:

jupyter lab --version

  • This will show jupyter lab installed version.

Launching Jupyter Lab#

  • We can launch Jupyter lab the same way we explained above, you can change the working directory first then launch jupyter lab from there this way:

jupyter lab

  • Note that you can pass the working directory as an argument to this command, however, I try to follow the easiest way, the advanced way is left for advanced courses about Jupyter lab or notebook.

Note#

We still missing one step, we cannot run R from jupyter notebook or lab yet. This is explained in the next section.

Installing the R Kernel#

  • In this section, we need to install the R kernel, which is the core of R programming language, and without it we cannot run R code from Jupyter.

  • A word of caution, We can do this from RGui Application, but sometimes it does not succeed, thus, we need to start R from the command prompt by typing R. If it does not start R, it means R is not the in the path, please go back to first lesson and add follow the steps to add it to the path.

  • Launch R from the command prompt and we need to install a package from there. This package is named IRkernel (please watch out the spelling, because it matters a great deal, if it does not work, check the spelling first)

  • To install the R Kernel you can run the following command from the R Console. source

> install.packages('IRkernel')

  • We still lack one step, I will train to explain things to you in more details, do not take my word for it, start thinking as a programmer from here on, yes you are programmer, I mean it.

Registering the kernel in the current R installation#

  • After installing the IRkernel, we need to add to Jupyter which can be achieved by installing a kernel spec using installspec() function from the IRkenel package. This allows Jupyter to recognize the kernel and work with it interactively:

  • As a programmer, or a future programmer, you don’t want to take everything for granted. If you are new to R at all (and I suppose you are), what we are going to do here will not be clear to you, but this concerns the installation on the system more than the language.

  • Here will try to understand what the installspec() does when calling it, and can customize our Rkernel or accept the defaults if you are happy with them.

  • In the next piece of code, we will load the IRkernel to the working environment, then call another function args() to display the arguments of the installspec() function. You can alternatively run the help(installspec() for more information. However, here I will explain that to you.

  • Getting the help of what you are doing:

> library("IRkernel")
> args(installspec)
function (user = NULL, name = "ir", displayname = "R", rprofile = NULL, 
    prefix = NULL, sys_prefix = NULL, verbose = getOption("verbose"))

  • user = NULL: to install the kernel for the current user only. You can set this argument to user=FALSE to install the kernel system wide.

  • name = "ir": This indicates the name of the kernel.

  • displayname = "R": This the name which is displayed in the notebook (default: “R”)).

  • Now it is time to make Jupyter aware of R kernel, I will accept the default except for the displayname which I will add The version (This makes me aware of what version I am working with, also it will be help debug some errors when they occur)

installspec(displayname = "R 4.3") # R 4.3 is the latest version of writing this lesson

  • You will see some programmers using this in one step by using what is called namespace, which means we call installspec() function directly from where it locates without the need to load the IRkernel package. I did it in two steps for the purpose of education, you can do it this way as well:

> IRkernel::installspec(displayname = "R 4.2")

Note#

By not setting different arguments, multiple calls will overwrite the kernel with a kernel spec pointing to the last R interpreter you called that commands from.

Installing R Versions (optional and Advanced)#

  • You can install kernels for multiple versions of R by supplying a name and displayname argument to the installspec() call (You still need to install these packages in all interpreters you want to run as a jupyter kernel!):

# in R 3.3
IRkernel::installspec(name = 'ir33', displayname = 'R 3.3')
# in R 3.2
IRkernel::installspec(name = 'ir32', displayname = 'R 3.2')

Something Extra#

  • You can install R from github using these commands. The first command installs the necessary libraries.

  • The devtools is the library used to install IRkernel from the its repository on github.

  • The last command will register the kernel for jupyter.

install.packages(c('repr', 'IRdisplay', 'pbdZMQ', 'devtools'))
devtools::install_github('IRkernel/IRkernel')
IRkernel::installspec()  # to register the kernel in the current R installation

Installing shortcuts#

  • Make useful shortcuts available: If you use Jupyter lab (and you should!), install Ryan Homer’s text-shortcuts extension:

jupyter labextension install @techrah/text-shortcuts

  • It will make available the familiar shortcuts Ctrl/⌘ + ⇧ + M (Ctrl + Shift + M keys) for PC users or (Command + shift + M) for Mac users to insert %>% and Alt + - to insert <-.

Conclusion#

  • Now your environment is set up correctly, you can start working on your projects using Jupyter notebook or jupyter lab. We have also seen how to install and set up RStudio IDE. Using RStudio or Jupyter depends on your need and the type of project you are working on, however, I’d recommend using RStudio if you are developing packages and writing code for reuse or working with multiple projects. It is also suitable if you develop a blog or writing a book.

  • Use Jupyter if you are a data scientist and need visualization to be in the same file (this is also possible from rmarkdown from RStudio). It is also great environment for education and for presentation.
